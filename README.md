# Лабораторная работа №22. Интерактивные веб-приложения на JavaScript

## 1. Описание проекта

Данный проект представляет собой интерактивный веб-калькулятор,
разработанный с использованием HTML, CSS и JavaScript.

Приложение позволяет пользователю выполнять базовые математические
операции через интерфейс с кнопками. Проект создан в рамках лабораторной
работы по разработке интерактивных веб‑приложений.

> [!NOTE]
> Проект демонстрирует использование ES6 модулей, объектно‑ориентированного подхода и разделения логики приложения.

Основные цели проекта:

- реализация пользовательского интерфейса калькулятора
- реализация логики вычислений
- организация кода с использованием классов
- обработка ошибок и валидация пользовательского ввода
- использование системы контроля версий Git и удаленного репозитория GitHub

# 2. Функциональность

## Поддерживаемые операции

Калькулятор поддерживает следующие операции:

- сложение +
- вычитание -
- умножение \*
- деление /
- работа с десятичными числами .
- очистка выражения C
- вычисление результата =

Пример выражения:

5 + 3 \* 2

Результат:

11

## Ограничения

Для предотвращения некорректного ввода реализованы следующие
ограничения:

- нельзя вводить два оператора подряд
- нельзя вводить несколько точек в одном числе
- выражение не может начинаться с оператора (кроме минуса)
- выражение не может заканчиваться оператором
- запрещено деление на ноль

> [!WARNING]
> Некорректные выражения (например 5++3 или 5.2.1) не будут добавлены в строку выражения. Это предотвращает ошибки вычислений.

## Обработка ошибок

Если пользователь вводит некорректное выражение, калькулятор выводит
сообщение "Ошибка".

При возникновении ошибки:

- вычисление не выполняется
- выражение очищается
- пользователь может начать новый ввод

> [!TIP]
> После появления ошибки достаточно нажать кнопку C, чтобы полностью очистить состояние калькулятора.

## Класс Calculator

Класс Calculator реализует основную логику работы калькулятора.

Основные поля класса:

display
ссылка на элемент дисплея

expression
строка, содержащая текущее выражение

hasError
флаг состояния ошибки

Основные методы:

init()
инициализация калькулятора

handleButtonClick(value)
обработка нажатий кнопок

addToExpression(value)
добавление символа в выражение с проверкой корректности

calculate()
вычисление результата

safeEvaluate(expr)
безопасное выполнение вычисления

clear()
очистка выражения

updateDisplay()
обновление дисплея

showError()
отображение ошибки

> [!NOTE]
> Логика вычислений вынесена в отдельный класс, что улучшает читаемость кода и облегчает поддержку проекта.

## Схема взаимодействия компонентов

Работа приложения происходит следующим образом:

Пользователь нажимает кнопку->

Обработчик события передает значение кнопки->

Метод handleButtonClick определяет действие->

Выполняется добавление символа или вычисление->

Обновляется дисплей

# 4. Логика работы

## Последовательность действий при нажатии кнопок

1.  Пользователь нажимает кнопку калькулятора
2.  Обработчик события получает значение кнопки
3.  Метод handleButtonClick анализирует действие
4.  Выполняется добавление символа или вычисление
5.  Обновляется дисплей

## Валидация ввода

Перед добавлением символа выполняются проверки:

- отсутствие двух операторов подряд
- отсутствие нескольких точек в числе
- корректное начало выражения
- корректное окончание выражения

Если ввод некорректен, символ не добавляется.

> [!TIP]
> Такая проверка называется клиентской валидацией и выполняется непосредственно в браузере.

## Выполнение вычислений

При нажатии кнопки "=":

1.  выражение проверяется на корректность
2.  выполняется вычисление
3.  результат выводится на дисплей
4.  результат становится новым выражением

> [!WARNING]
> Для вычислений используется eval(), поэтому перед выполнением выражение проходит проверку безопасности.
