# Лабораторная работа №22. Интерактивные веб-приложения на JavaScript

## 1. Описание проекта

Данный проект представляет собой интерактивный веб-калькулятор,
разработанный с использованием HTML, CSS и JavaScript.

Приложение позволяет пользователю выполнять базовые математические
операции через интерфейс с кнопками. Проект создан в рамках лабораторной
работы по разработке интерактивных веб‑приложений.

> [!NOTE]
> Проект демонстрирует использование ES6 модулей, объектно‑ориентированного подхода и разделения логики приложения.

Основные цели проекта:

- реализация пользовательского интерфейса калькулятора
- реализация логики вычислений
- организация кода с использованием классов
- обработка ошибок и валидация пользовательского ввода
- использование системы контроля версий Git и удаленного репозитория GitHub

---

# 2. Функциональность

## Поддерживаемые операции

Калькулятор поддерживает следующие операции:

- сложение +
- вычитание -
- умножение \*
- деление /
- работа с десятичными числами .
- очистка выражения C
- вычисление результата =

Пример выражения:

5 + 3 \* 2

Результат:

11

## Ограничения

Для предотвращения некорректного ввода реализованы следующие
ограничения:

- нельзя вводить два оператора подряд
- нельзя вводить несколько точек в одном числе
- выражение не может начинаться с оператора (кроме минуса)
- выражение не может заканчиваться оператором
- запрещено деление на ноль

> [!WARNING]
> Некорректные выражения (например 5++3 или 5.2.1) не будут добавлены в строку выражения. Это предотвращает ошибки вычислений.

## Обработка ошибок

Если пользователь вводит некорректное выражение, калькулятор выводит
сообщение "Ошибка".

При возникновении ошибки:

- вычисление не выполняется
- выражение очищается
- пользователь может начать новый ввод

> [!TIP]
> После появления ошибки достаточно нажать кнопку C, чтобы полностью очистить состояние калькулятора.

## Класс Calculator

Класс Calculator реализует основную логику работы калькулятора.

Основные поля класса:

display
ссылка на элемент дисплея

expression
строка, содержащая текущее выражение

hasError
флаг состояния ошибки

Основные методы:

init()
инициализация калькулятора

handleButtonClick(value)
обработка нажатий кнопок

addToExpression(value)
добавление символа в выражение с проверкой корректности

calculate()
вычисление результата

safeEvaluate(expr)
безопасное выполнение вычисления

clear()
очистка выражения

updateDisplay()
обновление дисплея

showError()
отображение ошибки

> [!NOTE]
> Логика вычислений вынесена в отдельный класс, что улучшает читаемость кода и облегчает поддержку проекта.

# 4. Логика работы

## Последовательность действий при нажатии кнопок

1.  Пользователь нажимает кнопку калькулятора
2.  Обработчик события получает значение кнопки
3.  Метод handleButtonClick анализирует действие
4.  Выполняется добавление символа или вычисление
5.  Обновляется дисплей

## Валидация ввода

Перед добавлением символа выполняются проверки:

- отсутствие двух операторов подряд
- отсутствие нескольких точек в числе
- корректное начало выражения
- корректное окончание выражения

Если ввод некорректен, символ не добавляется.

> [!TIP]
> Такая проверка называется клиентской валидацией и выполняется непосредственно в браузере.

## Выполнение вычислений

При нажатии кнопки "=":

1.  выражение проверяется на корректность
2.  выполняется вычисление
3.  результат выводится на дисплей
4.  результат становится новым выражением

> [!WARNING]
> Для вычислений используется eval(), поэтому перед выполнением выражение проходит проверку безопасности.

# Лабораторная работа №23. Интерактивные веб-приложения на JavaScript

## 1. Описание проекта

Закрепить и систематизировать знания по
JavaScript путём разработки интерактивных веб-приложений с использованием HTML,
CSS и JavaScript, а также закрепить навыки работы с системой контроля версий Git и
удалёнными репозиториями GitHub.

> [!NOTE]
> Игра реализована с использованием HTML, CSS и JavaScript и работает полностью в браузере.

### Правила игры

1.  В игре участвуют два игрока: X и O.
2.  Игроки делают ходы по очереди.
3.  Игрок X всегда начинает игру.
4.  Один ход занимает одну свободную клетку.
5.  Побеждает игрок, который первым соберет линию из трех одинаковых
    символов.

### Логика победы и поражения

Победа фиксируется, если один из игроков заполняет:

- горизонтальную линию
- вертикальную линию
- диагональную линию

Если поле заполнено полностью и победитель не найден - объявляется
ничья.

> [!WARNING]
> После победы или ничьи игра останавливается и новые ходы невозможны.

### Особенности реализации

- игровое поле реализовано с помощью CSS Grid
- состояние игры хранится в массиве JavaScript
- каждая клетка имеет индекс
- обработчики событий реагируют на клики пользователя

---

# 2. Игровая механика

## Как определяется победитель

После каждого хода выполняется проверка всех возможных выигрышных
комбинаций. Если три клетки содержат одинаковый символ, объявляется
победитель.

## Как обрабатываются клики

При клике на клетку выполняется функция handleCellClick:

1.  определяется индекс клетки
2.  проверяется, свободна ли она
3.  проверяется, активна ли игра
4.  записывается символ текущего игрока
5.  обновляется массив состояния
6.  выполняется проверка результата

> [!TIP]
> Если клетка уже занята, клик игнорируется.

## Как работает перезапуск

Кнопка Restart запускает функцию restartGame.

Функция:

- очищает массив board
- устанавливает текущего игрока X
- активирует игру
- очищает визуальные клетки

После этого начинается новая игра.

---

# 3. Структура данных

## Массив board

Массив board хранит состояние игрового поля.

Пример:

board = ["","","","","","","","",""]

Каждый элемент массива соответствует клетке.

Когда игрок делает ход, в массив записывается X или O.

> [!NOTE]
> Массив board является основной моделью состояния игры.

## Условия победы

Выигрышные комбинации хранятся в массиве:

winningConditions = [ [0,1,2], [3,4,5], [6,7,8], [0,3,6],
[1,4,7], [2,5,8], [0,4,8], [2,4,6]]

Каждый набор индексов представляет линию победы.

## Хранение состояния игры

Состояние игры хранится в переменных:

currentPlayer - текущий игрок (X или O)

gameActive - активна ли игра

board - состояние клеток

---

# 4. Алгоритмы

## Алгоритм проверки победы

1.  перебрать все выигрышные комбинации
2.  получить значения клеток из board
3.  проверить совпадение символов
4.  если совпадают - объявить победителя
5.  остановить игру

## Алгоритм определения ничьи

1.  проверить наличие пустых клеток
2.  если пустых клеток нет
3.  и победитель не найден
4.  объявляется ничья

> [!WARNING] Ничья возможна только при полностью заполненном поле.

## Алгоритм смены игрока

Если победы нет:

1.  проверить текущего игрока
2.  если X → следующий игрок O
3.  если O → следующий игрок X
4.  обновить статус игры
